# WebX Working Group

The group of people that write the spec and decide the future of WebX.\
Members are decided by getting one person that represents each browser and DNS with significant market share that tries to follow the specification.\
Decisions made by the group must have over 50% in favor to be considered for addition, each member has one vote that is equal to others, except the case where there is dissagrement, then members representing the area discussed have priority talk.\
All communications from members are private by default, any member may disclose what has been talked in a meeting.

## Feature Consideration

Each new features is discussed for two main goals: It fills a gap that can't be filled and it preserves the privacy of the user to prevent the current issues of the normal web.

Current Considered:

- File downloads
- WebSockets
- SSE
- Canvas
